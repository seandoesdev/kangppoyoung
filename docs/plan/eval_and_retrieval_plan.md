# 평가 신뢰화(N회 평균) + Retrieval 개선 계획

> 배경: 청크 병합으로 평균 53.8→56.4(최고)·오답 3→1, 특히 3번 절차 55→96. 다만 **1회 스냅샷은
> 노이즈 ±15~40**이라 회귀(1·10)·미해결(4번)이 노이즈인지 실측인지 분별 불가.
> 순서: **(A) N회 평균 채점으로 현재(병합) 성능 확정 → (B) 그 결과로 retrieval 정조준.**

## Phase A — N회 평균 채점 하니스

### 목표
합성기(gpt-4o) 비결정성으로 인한 항목별 ±15~40 노이즈를, 항목당 **N회(기본 10) 독립 실행 평균**으로
줄여 **신뢰 가능한 성능 수치(mean ± std)**를 만든다. 이후 모든 튜닝/회귀 판단의 기준.

### 설계
- **단일 소스**: `eval/eval_set.json`(골든 [`search-eval.md`](../eval/search-eval.md)에서 추출 — 질문·기대·기준).
- **수집**(`eval/collect.py`): 각 질문을 `POST /api/v1/search`로 **N회** 호출(검색은 결정적, 합성만 변동 →
  N개 답변). 동시 호출로 단축. evidence 문서 분포도 함께 수집.
- **채점**(워크플로우): **항목당 1 에이전트**가 그 질문의 N개 답변을 정답 판정 기준으로 각각 0~100 채점
  → run별 {verdict, score, multiSource}. (10 에이전트, 1회 1문항 N답변.)
- **집계**: 항목 mean±std, 전체 평균, 다출처율 → `docs/eval/results-<날짜>-avgN.md`.

### 산출
- 병합 corpus의 **신뢰 baseline**(mean±std). 회귀(1·10)가 실회귀인지 노이즈인지, 4번이 정말 0인지 확정.

## Phase B — Retrieval 개선 (Phase A 결과로 정조준)

### 가설(4번 성장단계)
병합으로 기초가이드에 성장단계별 자금 청크가 **생겼으나 검색에서 안 끌려옴**. 원인 후보:
1. **문서 균형/게이트**(P1 0.75·3)가 기초가이드 청크를 top-K에서 제외(변경공고 물량).
2. 병합 청크의 `embedding_text`가 질의 토큰("성장단계")과 어휘 불일치.
3. 질의 분석(QueryAnalyzer) 검색어가 성장단계 자금명을 포함 못함.

### 진단 → 조치(택1·조합, Phase A 후 확정)
- 진단: 4번 질의로 retrieval 후보의 문서·청크 분포를 직접 덤프(어느 청크가 왜 빠지는지).
- 조치 후보: P1 게이트/쿼터 재튜닝, 질의 확장(성장단계↔자금명), 병합 청크 embedding_text 보강.
- **각 조치는 N회 평균 하니스로 검증**(노이즈 분리).

### 회귀(1·10)
N회 평균으로 실회귀 확인 시: 병합 입도(decider 프롬프트) 조정 또는 retrieval에서 초점 청크 보존.

## 진행
1. (이번) Phase A 하니스 구현 + 10회 실행 → 병합 corpus 신뢰 baseline 기록.
2. 결과 기반 Phase B 진단 → 조치 1개 구현 → N회 재측정.
