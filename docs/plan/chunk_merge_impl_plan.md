# 청크 품질 개선 스테이지 — LLM 판단 + 결정론적 구성 (설계 v2)

> 상위: [`plan_overview.md`](plan_overview.md) P4. 구현 전 설계.
> 기준선: [`../eval/results-2026-06-20-baseline.md`](../eval/results-2026-06-20-baseline.md).
>
> **v2 변경**: ① 오프라인 모드 제거(온라인 전용, "한 번 정확하게" 1회 처리) ② 병합뿐 아니라 **분할**도 포함
> ③ LLM은 **경계 판단만**, 실제 청크 구성은 **프로그래머틱**으로 만들어 결정론적으로 확정.

PDF→XML 청킹 후, 청크를 **"업무적으로 유효한 가장 작은 의미 단위"**(절차면 단계 전부가 한 청크)로
다시 만든다. LLM은 어디서 단위가 나뉘는지 **판단만** 하고, 텍스트는 한 글자도 쓰지 않는다.

## 0. 핵심 원칙

- **온라인 전용**: 오프라인 폴백 없음. `OPENAI_API_KEY` 필수. 정확도 우선의 1회성 enrichment.
- **LLM은 판단만**: 단위 경계(병합/분할)를 판단. 텍스트 작성·구성은 절대 하지 않음.
- **결과는 프로그래머틱·결정론**: LLM 경계 결정을 받아 **코드가 무손실 span 연산으로 청크를 확정**.
  경계 결정은 content-hash로 캐시 → 재실행 byte-identical(결정론).

## 1. 목표 · 성공 기준

- **목표**: 청크 = 업무적으로 유효한 가장 작은 완결 단위(절차/목록/조건 통째).
- **성공**(eval 재채점): 절차/목록형(3·4번) 상승, evidence 단위 완결성↑, 런타임 보정(부모확장·2-hop) 의존↓, 회귀 없음.

## 2. 통합 모델 — 원자 재분할 (병합 + 분할을 하나로)

핵심 재설계. 병합과 분할을 **한 메커니즘**으로 처리한다.

1. **원자화(Atomize, 결정론)**: 입력 청크들을 결정론적 규칙으로 **최소 원자 span**(문장 / 줄 / 리스트 항목 /
   표 행 — 마커·구두점·줄바꿈 기준)으로 분해. **원자들의 순서 연결 = 원본**(무손실 타일링).
2. **경계 판단(LLM, 판단만)**: 원자 시퀀스 위에서 **단위 경계가 어디인가**를 LLM이 판단.
   - 서로 다른 원본 청크의 원자들 사이에 경계 없음 → **병합**.
   - 한 원본 청크 내부 원자들 사이에 경계 있음 → **분할**.
3. **그룹 구성(프로그래머틱, 결정론·무손실)**: 코드가 경계 사이 원자 span을 **순서대로 연결**해 최종 청크 확정.
   chunk_id·관계·embedding_text 재계산.

→ 병합·분할이 모두 "경계 판단 + 원자 그룹핑"으로 통일되고, **LLM은 경계만 / 구성은 코드**.

## 3. LLM 경계 판단 (판단만)

- **입력(컨텍스트 윈도)**: 현재까지 묶은 그룹 요약 + `heading_path` + 다음 원자(들).
  → "다음 원자가 이 단위를 계속 잇는가, 새 단위의 시작인가?"
- **출력**: 원자 사이 경계 boolean 시퀀스(또는 경계 인덱스). **텍스트 생성 없음.**
- **강제 경계(과병합 방지)**:
  - `heading` 경계는 기본 분리(새 소제목 = 새 단위).
  - 누적 크기 `max_chunk_chars`(기존 800) 초과 시 강제 경계.
- **배치**: `heading` 섹션 단위로 한 번에 경계 판단(호출 수 절감).

## 4. 프로그래머틱 구성 (결정론·무손실 — 핵심)

- LLM 경계 결정(원자 경계 위치)을 받아, 코드가 **경계 사이 원자 span을 순서대로 연결**해 청크 텍스트를 확정.
- **분할도 동일**: 한 원본 청크의 원자들이 경계로 나뉘면 코드가 각 그룹을 **별도 청크**로 만든다(텍스트는 원자
  span 그대로, 자르는 위치만 LLM이 고른 원자 경계).
- **무손실**: 모든 원자가 정확히 한 청크에 한 번 들어감(누락·중복·재작성 0) → 전체 연결 = 원본.
- **결정론**: 경계 결정이 같으면 구성 결과는 byte-identical(순수 함수). 경계 결정은 캐시(§6).

## 5. 검증 패스 (Pass 2, LLM 판단만)

- 각 구성 청크가 **단일·완결·크기 적정** 단위인지 LLM이 판단(검증). 절차면 단계 누락/순서, 끊긴 문장 점검.
- **실패 시**: 비차단 플래그(`review_required` + 사유). 보정은 **재경계(코드)** 로, LLM은 판단만.

## 6. 결정론 · 캐시 (오프라인 폴백 대신)

- LLM 호출은 비결정적 → **경계·검증 판단을 content-hash 키로 캐시**. 같은 입력 → 캐시 → 동일 출력.
- 구성은 순수 코드 → 결정만 주어지면 byte-identical.
- 크로스-머신 재현을 위해 **결정 로그(캐시)를 산출물과 함께 버전관리** 권장.
- 파이프라인 결정론은 이제 **"캐시된 LLM 결정 하의 결정론"**으로 정의(오프라인 폴백 없음은 설계 결정).

## 7. 무손실 보장 (verify 게이트)

- 원자화가 원본을 정확히 타일링(문자 손실 0) + 구성이 모든 원자를 1회 사용 → 연결 = 원본.
- `verify`: 보존율 ≥0.99·round-trip **미통과 시 병합 결과를 적용하지 않음**(안전).

## 8. 범위 · 한계

- **병합 + 분할** 모두(원자 재분할로 통합). 과소청킹(파편)뿐 아니라 과대·혼합 청크도 원자 경계로 분할.
- **한계**: 이미지/순서도 등 **미추출 콘텐츠는 원자가 없어 복구 불가** → Vision OCR(키 필요) 별개 과제.

## 9. 파이프라인 변경 · downstream

- **신규**: `pipeline/merge.py`(원자화 + 경계 호출 + 프로그래머틱 구성 + 검증), `providers`에 OpenAI 경계/검증
  클라이언트 + 캐시. (오프라인 provider 없음.)
- **위치**: `build`(청크 형성) 직후 ~ `relations`/`embedding`/`serialize` 직전. 재계산 정상 흐름.
- `config`: 스테이지 on/off, `max_chunk_chars`(기존), 캐시 경로.

## 10. 프롬프트 스케치

- **경계**: "정책자금 문서의 연속 원자 시퀀스다. [현재 단위 요약]. 다음 원자가 같은 '업무적으로 유효한 단위'를
  잇는가, 아니면 새 단위 시작(경계)인가? 각 위치 boolean만."
- **검증**: "이 청크가 단일·완결 업무 단위인가? 절차면 단계 누락/순서·끊긴 문장 점검. valid + 사유."
- 둘 다 구조화 출력(boolean 배열 / record).

## 11. 리스크 · 완화

| 리스크 | 완화 |
|---|---|
| 경계 오판(병합/분할) | 컨텍스트 윈도(그룹+heading) + 검증 패스 자기점검 |
| 과병합/과분할 | heading·크기 강제 경계, 검증 후 재경계 |
| 결정론(캐시 미스/머신차) | 결정 로그 버전관리, 구성은 순수 함수 |
| 키 필수(오프라인 없음) | 1회성 정확 처리로 수용(명시적 설계 결정) |
| 보존율 위반 | 원자 타일링 + verify 게이트(미통과 시 미적용) |
| 재임베딩 비용 | 일회성 |

## 12. 테스트

- **단위(LLM 없이)**: 원자화 무손실(연결 = 원본). **경계 결정을 mock**으로 주입했을 때 구성이
  결정론·무손실인지. 병합(연속 절차 원자 → 1청크)·분할(혼합 청크 원자 → 2단위) 케이스.
- **파이프라인**: 골든 문서 재청킹 후 `verify` 보존율·round-trip pass. '다. 융자절차' 섹션이 1청크인지.
- **평가**: `search-eval` 재채점(3·4번·다문서) 베이스라인 대비(노이즈 완화 위해 N회 평균 병행).

## 13. 단계 · 운영 · 롤백

1. **원자화 + 경계 인터페이스 + 프로그래머틱 구성 + 단위테스트**(경계 mock으로 결정론·무손실 검증 — LLM 불필요).
2. **OpenAI 경계/검증 provider + content-hash 캐시**.
3. **골든 문서 재청킹 → `verify`** 통과 확인.
4. **재적재 + `search-eval` 회귀** 측정.

- **재적재 필요**(청크·임베딩 변경). 키 필수(오프라인 폴백 없음).
- **롤백**: `config`로 스테이지 off → 기존 청킹.
